# 网站信息与工具清单（规划）

本站需呈现的**信息**与**工具**的完整清单。先规划，后实施。

---

## 〇、用户痛点与工具映射（总览）

用户从「拿到一瓶肽」到「用完、复购、选供应商」全流程中的痛点，与现有/可选工具的对应关系。**不只当前这几样工具**，下面列出可扩展项。

### 用户旅程与痛点

| 阶段 | 用户痛点 | 现有/规划中的工具 | 可补充的工具 |
|------|----------|-------------------|--------------|
| **拿到一瓶肽** | 不知道加多少水、配成多少浓度 | 复溶与剂量计算器 | — |
| | 不知道抽多少、打多少单位（尤其不同规格注射器） | 计算器给出 1 mL 注射器单位 | **注射器单位换算**：选 0.3/0.5/1 mL，输出对应 units |
| | 单位搞混：mcg / mg / mL / IU（部分肽） | 计算器内隐式换算 | **通用单位换算器**：mcg↔mg，可选 IU（如 HGH） |
| **复溶与储存** | 复溶后能放多久、怎么存 | Guide 文字、肽详情里的复溶说明 | **复溶稳定性/有效期**：按肽类型或选「常见 30 天 2–8°C」提示或简单「开瓶日期→建议用完日」 |
| **剂量与周期** | 一瓶能打几天、一个周期要几瓶 | 需手算 | **用量与周期计算器**：一瓶 mg + 单次剂量 mcg + 频率 → 可用天数；或目标周期 → 需几瓶 |
| **查肽与选型** | 不知道这个肽通常多少剂量、怎么用 | 肽目录 + 详情 | — |
| | 想对比两个肽（剂量、频率、用途） | 无 | **肽对比表/工具**：并排 2–3 个肽的剂量、频率、常见用途 |
| **纯度与信任** | 怕买到假/劣，不知道哪里验 | Purity & Verify 页 + Janoshik 外链 | — |
| | 手里有报告，不知道是不是真的 | 文案教「用 task ID 去官网验」 | **报告验证助手**：输入 task ID → 给出 Janoshik 验证页链接 + 步骤说明 |
| **采购与成本** | 不知道选哪家、谁有测试报告 | Suppliers 页（占位） | 供应商目录 + 报告链接 |
| | 想算「每剂量多少钱」 | 无 | **单次剂量成本**：输入一瓶价格 + 瓶内 mg + 单次剂量 mcg → 每 dose 成本（可选） |

### 痛点与工具汇总（按优先级）

| 优先级 | 痛点 | 工具 | 说明 |
|--------|------|------|------|
| **P0 已有** | 加多少水、抽多少、浓度多少 | 复溶与剂量计算器 | 核心，已有 |
| **P0 已有** | 这个肽是啥、通常多少剂量 | 肽目录 + 详情 | 已有 |
| **P0 已有** | 哪里验纯度、报告怎么信 | Purity & Verify 信息 + 外链 | 已有 |
| **P0 已有** | 想看分子长什么样 | 3D Structure 外链 | 已有 |
| **P1 建议补** | 注射器不是 1 mL（0.3/0.5 mL） | 注射器单位换算（或并入计算器） | 选 syringe 规格 → 显示对应 units |
| **P1 建议补** | 一瓶能打几天 / 一个周期要几瓶 | 用量与周期计算器 | 输入：瓶 mg、单次 mcg、频率 → 天数；或目标天数 → 瓶数 |
| **P1 建议补** | 有报告，想快速去验证 | 报告验证助手（task ID → 验证链接 + 步骤） | 纯前端 + 外链 |
| **P2 可选** | mcg/mg/mL 随手换算 | 通用单位换算器 | 小工具或计算器旁一栏 |
| **P2 可选** | 复溶后能用多久 | 复溶稳定性/有效期提示 | 按肽类型或通用「30 天 2–8°C」+ 开瓶日→建议用完日 |
| **P2 可选** | 两个肽并排对比 | 肽对比（并排 2–3 个） | 信息型，可做表格或简单选择器 |
| **P2 可选** | 每 dose 多少钱 | 单次剂量成本 | 输入单价+含量+单次剂量 → 每 dose 成本 |

结论：**工具不止当前三样**；在保留「纯前端、无后端」前提下，可增加**注射器单位、用量/周期、报告验证助手**等，并视需要再做单位换算、稳定性提示、肽对比、单次成本等。

## 〇·二、工具排序：使用频次 × 依赖程度 × 痛点程度

所有工具按**使用频次**（多常被用）、**依赖程度**（没它能否完成）、**痛点程度**（没有时多难受）三个维度打分（高/中/低），再给出**综合排序**（实施优先级）。

### 维度说明

| 维度 | 高 | 中 | 低 |
|------|----|----|-----|
| **使用频次** | 每次开瓶/每次打针或采购都会用 | 每个周期或换肽时用几次 | 偶尔查一次或好奇时用 |
| **依赖程度** | 没有就做不了或极易出错，必须用 | 没有能凑合（手算/猜），有则省事 | 纯锦上添花，没有也能完成 |
| **痛点程度** | 算错/验不到会直接影响安全或信任，很焦虑 | 不方便、要手算或多次查，有点烦 | 没有只是少一点方便或信息 |

### 分维度打分表

| 工具 | 使用频次 | 依赖程度 | 痛点程度 |
|------|----------|----------|----------|
| 复溶与剂量计算器 | 高（每瓶/每次调剂量） | 高（新手不敢动手，老手也要核对） | 高（算错=打错量） |
| 肽目录 + 详情 | 中高（查剂量/用法，新人反复查） | 高（不知道剂量就无从算） | 高（不知道剂量很焦虑） |
| 注射器单位换算 | 中高（用 0.3/0.5 mL 的人每次都要） | 中高（没有就得自己算或猜） | 高（抽错单位=打错量） |
| 用量与周期计算器 | 中（买前、开瓶后算 1～2 次） | 中（可手算） | 中高（算错会断药或买多） |
| Purity & Verify 信息 + 外链 | 中（采购前/拿到报告时看） | 中高（怕假的人非常依赖） | 高（不验就用的焦虑大） |
| 报告验证助手 | 低～中（拿到报告时用几次） | 中（有报告的人想快速验） | 中高（怕报告是假的） |
| 3D Structure 外链 | 低（好奇或写东西时看） | 低 | 低（纯增信/学习） |
| 通用单位换算器 | 低～中（偶尔查 mcg/mg） | 低（计算器里已含主要换算） | 低 |
| 复溶稳定性/有效期提示 | 低（开瓶时看一次） | 低（Guide 文字可替代） | 中（怕过期会看一下） |
| 肽对比 | 低（选型时用 1～2 次） | 低 | 中（想对比时没有要自己翻） |
| 单次剂量成本 | 低～中（比价/下单时用） | 低 | 中（想比价时没有要自己算） |

### 综合排序（按频次 × 依赖 × 痛点，高优先）

综合考虑「用得勤、离不开、痛点强」的排前面：

| 排名 | 工具 | 说明 |
|------|------|------|
| 1 | **复溶与剂量计算器** | 频次高、依赖高、痛点高；核心工具，必须优先做好 |
| 2 | **肽目录 + 详情** | 查剂量/用法的基础，没有就无从算；频次中高、依赖高 |
| 3 | **注射器单位换算** | 用非 1 mL 注射器的人每次都要，依赖与痛点都高；建议并入计算器或紧贴 |
| 4 | **用量与周期计算器** | 买几瓶、一瓶打几天，决策刚需；频次中、痛点中高 |
| 5 | **Purity & Verify 信息 + 外链** | 建立信任的入口，痛点高但频次不如计算类；保持并完善文案与外链 |
| 6 | **报告验证助手** | 有报告时快速验证，依赖与痛点中高，频次中 |
| 7 | **3D Structure 外链** | 频次与依赖都低，增信/学习用；保留即可 |
| 8 | **通用单位换算器** | 频次与依赖低，可作计算器旁小栏或独立小工具 |
| 9 | **复溶稳定性/有效期提示** | 频次低，Guide 可部分替代；可选 |
| 10 | **肽对比** | 选型时用，频次低；可选 |
| 11 | **单次剂量成本** | 比价时用，频次低～中；可选 |

### 实施建议（按排序）

- **先做稳 1～2**：复溶与剂量计算器、肽目录/详情（已有，可继续优化）。
- **接着补 3～4**：注射器单位（并入或紧贴计算器）、用量与周期计算器。
- **再补 5～6**：Verify 页文案与报告验证助手。
- **7～11**：按资源与需求决定是否做、何时做。

## 二、工具 (Tools)

### 1. 复溶与剂量计算器 (Reconstitution & Dosing Calculator) — `/tools/calculator`

**输入**

| 输入项 | 单位/说明 |
|--------|-----------|
| 肽质量 | mg（如 5 mg 一瓶） |
| 模式 A：按体积 | 稀释液体积 mL（如 2.5 mL BAC 水） |
| 模式 B：按目标浓度 | 目标浓度 mg/mL（如 2 mg/mL） |
| 剂量→体积 | 目标单次剂量 mcg（如 250 mcg） |
| 体积→剂量 | 拟抽取体积 mL（如 0.125 mL），用于反推剂量 |

**输出**

| 输出项 | 说明 |
|--------|------|
| 浓度 | mg/mL（模式 A 由质量÷体积算；模式 B 即目标浓度） |
| 模式 B 时 | 建议加入的稀释液体积 mL |
| 给定剂量对应的体积 | mL，以及「在 1 mL 注射器上的单位数」（1 mL = 100 units） |
| 给定体积对应的剂量 | mcg |

**交互**  
- 切换「按体积」/「按目标浓度」；  
- 所有输入可编辑，结果实时更新；  
- 页脚简短免责与储存/无菌提醒。

### 4. 注射器单位换算（建议补，可并入计算器）

**痛点**：用户用的不是 1 mL 注射器（常见 0.3 mL = 30 units、0.5 mL = 50 units），需要知道「抽多少 units」对应自己手里的注射器。

**输入**  
- 浓度 mg/mL（可与主计算器联动或手动输入）；  
- 目标剂量 mcg；  
- 注射器规格：0.3 mL / 0.5 mL / 1 mL（或 30 / 50 / 100 units）。

**输出**  
- 应抽取体积 mL；  
- 在该规格注射器上的 **units** 数。

**实现**：纯前端公式；可与现有复溶与剂量计算器做在同一页或子区块。

---

### 5. 用量与周期计算器（建议补）

**痛点**：一瓶能打几天？想打 4 周要买几瓶？

**输入**  
- 每瓶肽质量 mg；  
- 单次剂量 mcg；  
- 频率（如每日 1 次、每日 2 次、每周 1 次）；  
- 可选：目标使用天数（则输出需几瓶）。

**输出**  
- 一瓶可支撑天数；  
- 或：目标天数下所需瓶数（向上取整）。

**实现**：纯前端；可单独页 `/tools/vial-duration` 或并入 Calculator 页一栏。

---

### 6. 报告验证助手（建议补）

**痛点**：用户拿到 Janoshik 报告或 task ID，想一步跳转到「验证页」并看到简短步骤。

**输入**  
- Janoshik task ID（或报告链接，从中解析 ID）。

**输出 / 行为**  
- 生成 Janoshik 验证页链接（若官网支持带 ID 的 URL）或明确「打开 Janoshik 官网 → 输入 task ID」步骤；  
- 页面内简短步骤说明（如：1. 打开 janoshik.com 验证页 2. 输入下方 task ID 3. 核对报告内容）。

**实现**：纯前端 + 外链；不存报告内容。

---

### 7. 通用单位换算器（可选）

**痛点**：随手查 mcg ↔ mg、mL 与浓度下的质量等。

**输入**  
- 数值 + 单位对：如 250 mcg → mg；或 0.25 mL × 2 mg/mL → mg。

**输出**  
- 换算结果。  
可选：IU（如 HGH）与 mcg 的换算若采用固定系数，可做简单支持（需标注来源与免责）。

**实现**：纯前端；小工具或计算器页一侧。

---

### 8. 复溶稳定性 / 有效期提示（可选）

**痛点**：复溶后能放多久？开瓶日 → 建议用完日。

**输入**  
- 开瓶/复溶日期；  
- 可选：肽类型（不同肽可设不同「常见稳定性」如 30 天 2–8°C）。

**输出**  
- 建议在 X 日前用完；或通用提示「多数肽 2–8°C 下 30 天内使用，请以产品说明为准」。

**实现**：纯前端；静态规则或按肽类型查表；免责写清「以厂商/文献为准」。

---

### 9. 肽对比（可选）

**痛点**：BPC-157 和 TB-500 有啥区别？Semaglutide 和 Tirzepatide 剂量差多少？

**输入**  
- 用户选择 2–3 个肽（下拉或卡片）。

**输出**  
- 并排表格：名称、典型剂量、频率、常见用途/特点、复溶注意、PDB（有则链出）。

**实现**：纯前端；数据来自现有肽静态数据。

---

### 10. 单次剂量成本（可选）

**痛点**：这瓶 50 刀 5 mg，每次 250 mcg，一 dose 多少钱？

**输入**  
- 单瓶价格（用户输入）；  
- 瓶内含量 mg；  
- 单次剂量 mcg。

**输出**  
- 每 dose 成本（如 $2.50/dose）；可选：每瓶可打次数。

**实现**：纯前端；不存价格，仅当场计算。

## 六、小结：痛点与工具不止「当前这几样」

- **用户痛点**：覆盖从「拿到一瓶肽」到「复溶、打多少、能打几天、验报告、选供应商、算成本」全流程；不只「不会配、不会算、怕假货」。
- **已有工具**：复溶与剂量计算器、肽目录/详情、3D 外链、Purity & Verify 信息与外链。
- **建议补充（P1）**：注射器单位换算、用量/周期计算器、报告验证助手。
- **可选（P2）**：单位换算器、复溶稳定性提示、肽对比、单次剂量成本。
- 以上均可**纯前端、无后端**实现。

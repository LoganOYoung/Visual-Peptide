# 十类用户角色 × 需求痛点 × 网站检测

假设你是以下用户，带着真实需求和痛点到站，看网站能否解决问题。

---

## 1. 科研助理 / 实验室技术员

**身份**：在实验室按 SOP 做肽复溶与给药，每天要算浓度、体积、剂量。

**需求**：快速得到「加多少稀释液、最终浓度、每次抽多少 mL/多少单位」，不能算错。

**痛点**：手算容易出错；不同肽、不同规格的瓶子要反复算；希望有统一工具，不用自己写公式。

**检测网站**：
- **首页** → 看到 “Precision without the math”、工具卡片，能直接点进 **Recon & Dosing Calculator**。
- **Tools → Recon & Dosing Calculator**：输入肽质量(mg)、稀释液(mL)、目标剂量(mcg)、注射器规格(0.3/0.5/1 mL)，得到浓度、需抽取体积、单位数。✅ **能解决** 核心计算需求。
- **Guide**：复溶步骤、浓度与剂量概念，可作 SOP 补充。✅
- **结论**：能解决。建议在首页或 Tools 页用一句话点出「适合实验室/按协议给药」会更贴合身份。

---

## 2. 健身 / 抗衰爱好者（自我研究）

**身份**：对 BPC-157、TB-500、Ipamorelin 等有研究兴趣，想自己搞清剂量和用法。

**需求**：查某肽的典型剂量、频率、复溶注意；不想被供应商话术带着走，想有中立参考。

**痛点**：网上信息杂、单位混用(mcg/mg/IU)；不知道「一瓶 5 mg 能打几次」；怕剂量搞错。

**检测网站**：
- **首页** → 有 Peptide Library、Popular peptides 入口。✅
- **Peptides → Library**：按 Metabolic / Repair & Healing / Cognitive 分类，点进 **BPC-157** 等详情页，有 typical dose、frequency、recon notes。✅ **能解决** 剂量与用法参考。
- **Tools → Vial & Cycle**：算一瓶用几天、要买几瓶。✅
- **Guide + FAQ**：复溶步骤、常见问题。✅
- **结论**：能解决。Library 和工具链清晰；若在肽详情页加「用计算器算我这瓶」的快捷入口会更顺。

---

## 3. 医生 / 临床研究者

**身份**：做代谢或相关研究，接触 GLP-1 类等肽，需要严谨的剂量与单位换算。

**需求**：快速查 Semaglutide / Tirzepatide 等剂量范围、单位换算(mcg ↔ mg)、文献向的严谨表述。

**痛点**：临床剂量与「研究用」剂量表述不一；需要可靠换算，避免用药错误。

**检测网站**：
- **Peptides** → **Semaglutide / Tirzepatide** 等：有 typical dose、frequency、half-life、CAS、分子量。✅ **能解决** 快速参考。
- **Tools → Unit Converter**：mcg ↔ mg 等换算。✅
- **Tools → Recon & Dosing Calculator**：按 mg/vial、diluent、dose 算体积与单位。✅
- **About / 免责**：研究/教育用途说明，符合「不替代临床决策」的定位。✅
- **结论**：能解决。站点明确 research-only，工具和资料能满足严谨换算与参考；若增加「文献/临床剂量差异」类说明会更贴合医生顾虑。

---

## 4. 采购 / 质控（验证报告真伪）

**身份**：负责采购或质控，拿到供应商给的 Janoshik 报告，要确认报告没被篡改。

**需求**：用 Task ID + 验证码在官网校验，确认报告真实、对应批次。

**痛点**：假报告、P 图报告多；不会用 Janoshik 官网或不知道有验证功能。

**检测网站**：
- **首页** → CTA「Verify before you source」→ **Verify** 页。✅
- **Verify (Purity & Verify)**：说明 Janoshik、HPLC/质谱/效价；**Report Verifier** 组件：输入 Task ID 等验证。✅ **能解决** 报告真伪验证。
- **Peptide Transparency Board**：近期验证的批次列表，增强「先验证再采购」心智。✅
- **结论**：能解决。验证流程和入口明确；若在 Verify 页注明「适合采购/质控核对」会更易被这类用户发现。

---

## 5. 学生 / 自学者（零基础）

**身份**：刚开始接触肽，不懂复溶、浓度、剂量概念。

**需求**：从零学起：什么是复溶、浓度怎么算、剂量 mcg 是什么意思、步骤怎么做。

**痛点**：公式和单位看不懂；搜到的内容要么太学术要么太零散；需要一条龙入门。

**检测网站**：
- **Help → Guide**：复溶步骤、所需材料、浓度与剂量概念、链接到计算器。✅ **能解决** 入门概念。
- **Help → FAQ**：常见问题集中回答。✅
- **Tools → Calculator / Syringe Planner**：在理解概念后，用工具巩固「浓度→体积→剂量」。✅
- **结论**：能解决。Guide + FAQ + 工具形成闭环；若在 Guide 开头加「零基础阅读路径」或目录会更友好。

---

## 6. 多肽对比选择者（在几种肽之间纠结）

**身份**：想从 BPC-157、TB-500、Ipamorelin 等里选一个或组合，需要对比剂量、频率、复溶。

**需求**：并排看 2–3 个肽的剂量、频率、说明、复溶注意，辅助决策。

**痛点**：多个标签页来回切；信息分散，不好比。

**检测网站**：
- **Peptides → Compare**：选最多 3 个肽，表格并排显示 name、description、typical dose、frequency、recon、category 等。✅ **能解决** 对比需求。
- **结论**：能解决。对比页是明确差异化功能；若在 Library 或详情页加「加入对比」入口会提高发现率。

---

## 7. 成本敏感用户（算每针多少钱、一瓶用多久）

**身份**：想控制预算，算「每针成本」「一瓶能用多久」「一个周期要几瓶」。

**需求**：输入单价、规格、剂量，得到每针成本；或输入剂量与频率，得到一瓶天数、所需瓶数。

**痛点**：自己算容易漏项或单位搞混；不同肽、不同规格要反复算。

**检测网站**：
- **Tools → Cost per Dose**：价格/瓶、mg/瓶、剂量(mcg) → 每针成本。✅ **能解决** 单次注射成本。
- **Tools → Vial & Cycle**：一瓶用几天、目标周期要几瓶。✅ **能解决** 用量与采购量。
- **结论**：能解决。两个工具直接对应痛点；可从首页或 Tools 页用「省钱/规划用量」类文案引导。

---

## 8. 视觉 / 动手型用户（抽象数字看不懂）

**身份**：看「抽 0.05 mL」没概念，希望看到「抽到注射器上哪一格」的图。

**需求**：输入复溶参数和剂量后，看到注射器示意图，标出应抽到的刻度/填充高度。

**痛点**：数字不直观；怕抽多或抽少；希望「所见即所抽」。

**检测网站**：
- **首页** 主推 **Syringe Planner**（Visual Syringe）。✅
- **Tools → Syringe Planner**：输入肽 mg、稀释液 mL、剂量 mcg、注射器规格 → **Visual Syringe** 显示注射器外观与填充液位。✅ **能解决** 直观看到「抽到哪里」。
- **结论**：能解决。Syringe Planner 是强差异化功能，首页已突出；若在计算器页加「想看注射器示意？去 Syringe Planner」链接会更闭环。

---

## 9. 结构 / 化学背景用户（想看 3D 和 PDB）

**身份**：做结构或化学相关，想快速看某肽的 3D 结构、PDB ID、分子量等。

**需求**：站内查看 3D 结构，或拿到 PDB 去 RCSB 进一步查；了解分子量等用于计算或文献。

**痛点**：PDB 分散在各处；希望一站内既能看结构又能跳转权威库。

**检测网站**：
- **3D Structure**：站内 3D 查看器，输入/选择 PDB（含 Semaglutide 6XBM 等），拖拽旋转、缩放；无对应肽时显示「Open in RCSB」。✅ **能解决** 站内看 3D + 外链 RCSB。
- **Peptide 详情页**：有 PDB 的显示 3D 区块；有 molecular weight、CAS、sequence 等。✅
- **结论**：能解决。3D 页与肽详情结合良好；若在 Library 列表里标出「有 3D」的肽会便于筛选。

---

## 10. 谨慎型用户（怕假货，先验证再买）

**身份**：非常在意真货与纯度，希望先知道怎么验证、有哪些第三方测试，再考虑购买。

**需求**：了解第三方检测（如 Janoshik）、如何用 Task ID 验证、有没有公开的测试结果可查。

**痛点**：不信任单方面说法；不知道有哪些可靠验证方式；希望看到「透明度」。

**检测网站**：
- **首页** CTA「Verify before you source」→ **Verify**。✅
- **Verify**：Janoshik 介绍、验证方式、**Report Verifier**、**Peptide Transparency Board**（近期验证批次）、public.janoshik.com 数据库链接。✅ **能解决** 「如何验证」和「看到他人验证结果」。
- **结论**：能解决。验证优先的动线清晰，Transparency Board 增强信任；若在 About 或 FAQ 再强调「我们不卖肽，只提供工具与验证入口」会更符合谨慎用户预期。

**整体结论**：十类用户的需求和痛点在当前站点都能找到对应功能；导航（Tools / Peptides / 3D / Verify / Help）和首页卡片已覆盖主要场景。可优化点：在关键页用一两句「给谁用」的文案（实验室、采购、零基础等）、在 Calculator 与 Library 间增加交叉入口（如「用计算器算这瓶」「看 3D」）、Compare 从 Library 或详情页更易发现。
